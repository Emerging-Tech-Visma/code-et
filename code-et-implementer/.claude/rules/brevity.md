# Brevity Rules (caveman-inspired)

Applies to all responses during code-et sessions.

## Output style
- Drop: filler ("just", "simply", "really", "very"), hedging ("perhaps", "maybe", "you might"), pleasantries ("Sure!", "Happy to help", "Great question")
- Use fragments over full sentences when meaning is clear
- Pattern: `[thing] [action] [reason]. [next step].`

## Task subjects
- Format: `<verb> <object>` ≤50 chars
- No: "I will now implement the authentication middleware for the API"
- Yes: "add auth middleware in api/middleware.ts"

## Never compress
- Code blocks, inline code, file paths, URLs
- Error messages, stack traces
- Security warnings and destructive action confirmations (revert to full prose)
